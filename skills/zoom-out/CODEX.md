# Zoom Out — версия для Codex CLI (GPT под капотом)

Это Codex-адаптация скилла `zoom-out`; логика/процесс те же, что в [SKILL.md](SKILL.md), здесь только дельта под Codex.

Скилл чисто инструкционный: одна короткая фраза-промпт («поднимись на уровень абстракции, дай карту модулей и вызывающих, в терминах доменного глоссария проекта»). Никаких субагентов, инструментов, скриптов или ресурсных файлов нет — поведение переносится **1-в-1, без изменений**. Единственная дельта под Codex — **где он живёт** и **как триггерится** (у Claude-скилла это делает `description`, а тут вдобавок стоит `disable-model-invocation: true`, то есть авто-вызова нет даже в Claude Code — только ручной).

## Куда положить и как активировать

Codex автоматически читает `AGENTS.md` (корень проекта или `~/.codex/AGENTS.md`). Авто-триггера по `description` (как у Claude-скиллов) у Codex нет, поэтому есть три способа:

1. **Слэш-вызов `/zoom-out`** — положи фразу как кастомный промпт в `~/.codex/prompts/zoom-out.md`. Тогда в чате набираешь `/zoom-out`, и Codex подставляет текст.
2. **Правило в `AGENTS.md`** — чтобы кодовая фраза вроде «zoom out» / «подними уровень абстракции» включала это поведение в любом проекте.
3. **Вручную** — просто вставить фразу в чат (скилл и так задуман как ручной, `disable-model-invocation: true`).

Готовый файл `~/.codex/prompts/zoom-out.md` (тело = текст промпта):

```md
Я плохо знаю этот участок кода. Поднимись на уровень абстракции выше.
Дай карту всех релевантных модулей и вызывающих (callers), используя
доменный глоссарий проекта.
```

Минимальный блок для `AGENTS.md`, если хочешь вызов кодовой фразой:

```md
## Zoom out (карта кода)
Когда пользователь просит «zoom out» / «подними уровень абстракции» —
не ныряй в детали реализации. Поднимись на уровень выше и выдай карту
релевантных модулей и их вызывающих, в терминах доменного глоссария проекта.
```

## Что меняется против Claude-версии

Ничего в поведении. Единственное отличие — носитель и триггер (промпт-файл `~/.codex/prompts/zoom-out.md`, правило в `AGENTS.md` или ручная вставка) вместо Claude-скилла с `description`. Субагентов, worktree, MCP-инструментов и bundled-скриптов скилл не использует — этих секций здесь нет за неприменимостью.
