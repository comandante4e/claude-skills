# Triage — версия для Codex CLI (GPT под капотом)

Это Codex-адаптация скилла `triage`. Логика и процесс — государственная машина ролей, три бакета «что требует внимания», шаги триажа конкретного issue, шаблоны agent brief / needs-info, `.out-of-scope/` — **те же, что в [SKILL.md](SKILL.md)**. Здесь только дельта механики под Codex.

## Куда положить и как активировать

Codex автоматически читает `AGENTS.md` (корень проекта или `~/.codex/AGENTS.md`). Авто-триггера по `description` (как у Claude-скиллов) у Codex **нет** — нужно либо правило в `AGENTS.md`, либо кастомный промпт.

Три способа активации, выбирай любой:
- **Слэш-команда `/triage`** — положи файл `~/.codex/prompts/triage.md` (тело — инструкции из `SKILL.md`, плюс дельта ниже). Тогда `/triage <запрос>` в чате Codex запускает воркфлоу, `<запрос>` подставится как аргумент.
- **Правило в `AGENTS.md`** — Codex подхватит его в каждой сессии проекта (блок ниже).
- **Вручную** — «прочитай `skills/triage/SKILL.md` и оттриажь #42».

Минимальный блок для `AGENTS.md`:

```md
## Triage (воркфлоу разбора issue)
Когда пользователь говорит «триаж», «оттриажь #N», «что требует внимания»,
«перевести #N в ready-for-agent» и т.п. — веди разбор по skills/triage/SKILL.md:
категория (bug/enhancement) + состояние (needs-triage / needs-info /
ready-for-agent / ready-for-human / wontfix), ровно одна роль каждого вида.
Каждый комментарий/issue при триаже начинай с дисклеймера
«> *This was generated by AI during triage.*».
Доступ к трекеру — через CLI трекера (для GitHub — `gh`); нет `gh`/доступа —
остановись и сообщи, не выдумывай состояние issue. Маппинг канонических ролей
на реальные лейблы должен быть выдан заранее. Полная версия — skills/triage/CODEX.md.
```

## Что меняется против Claude-версии

Скилл **не** использует субагентов и не опирается на harness-изоляцию — секций про worktree/субагентов здесь нет. Реальные зависимости — доступ к issue-трекеру, вызов смежного скилла `grill-with-docs` и чтение/запись файлов на диске. Дельта только в них.

### 1. Доступ к issue-трекеру — через CLI, который пользователь подключает сам
Скилл читает issue, комментарии, лейблы, постит комментарии, меняет лейблы и закрывает issue. Примеры в `SKILL.md` и брифах используют `gh` (GitHub CLI). У Codex это **не встроено**:
- агент вызывает `gh` (или CLI другого трекера) через shell — он должен быть установлен и авторизован (`gh auth status`);
- если трекер не GitHub или MCP-сервер трекера не подключён — настрой его в `~/.codex/config.toml` (секция `[mcp_servers.*]`) либо используй соответствующий CLI;
- **Guard:** нет `gh`/доступа к API трекера — **остановись и сообщи пользователю**, не угадывай состояние issue, лейблы и историю комментариев по памяти.

### 2. Слэш-вызов `/triage` и переданный запрос
В Claude Code `/triage <текст>` — слэш-команда скилла. В Codex эквивалент — кастомный промпт `~/.codex/prompts/triage.md`; натуральный запрос («покажи что требует внимания», «глянем #42», «move #42 to ready-for-agent») передаётся как аргумент промпта. Интерпретация запроса и сами шаги — без изменений.

### 3. Вызов смежного скилла `grill-with-docs` (шаг 4)
Claude-версия на шаге Grill зовёт `/grill-with-docs`. У Codex слэш-скиллов нет:
- если `grill-with-docs` тоже перенесён под Codex — запусти его как отдельный промпт `~/.codex/prompts/grill-with-docs.md` или прочитай его `SKILL.md` и проведи грилинг в текущей сессии;
- если такого скилла на машине нет — **не имитируй его молча**: проведи грилинг по здравому смыслу, но скажи пользователю, что канонического `grill-with-docs` нет. То же касается ссылки на `/setup-matt-pocock-skills` (выдаёт маппинг ролей на лейблы) — нет его, попроси маппинг у пользователя руками.

### 4. Ресурсные файлы и `.out-of-scope/` — переносятся как есть
- `AGENT-BRIEF.md` и `OUT-OF-SCOPE.md` переносятся **без изменений** и читаются Codex'ом с диска ровно так же — это просто справочные `.md`.
- `.out-of-scope/*.md` — обычная файловая база знаний: чтение на шаге Gather context и запись при `wontfix`-enhancement идут обычными файловыми операциями Codex, никакого харнесса не требуется.

### 5. GPT-специфика
- Дисклеймер `> *This was generated by AI during triage.*` — первой строкой **каждого** постящегося комментария/issue. Для GPT держи это явным пунктом чек-листа перед каждым постом.
- Инвариант ролей: ровно одна category-роль и одна state-роль; конфликт состояний — **остановись и спроси мейнтейнера**, не разрешай сам.
- Гейты-подтверждения не перепрыгивай: перед сменой роли / постом комментария / закрытием issue сначала озвучь, что собираешься сделать, и дождись согласия (особенно в Quick state override).
- Для воспроизводимых/скриптовых прогонов (например, «покажи все needs-triage батчем») используй `codex exec "<запрос>"` — неинтерактивно; интерактив оставь для самого разбора и грилинга.
