# Writing Skills — версия для Codex CLI (GPT под капотом)

Это Codex-адаптация скилла `write-a-skill`; логика и процесс те же, что в [SKILL.md](SKILL.md) — здесь только дельта под Codex.

Скилл **чисто инструкционный**: это набор правил, как писать новые скиллы (процесс из 3 шагов, структура папки, шаблон `SKILL.md`, требования к `description`, когда добавлять скрипты / разбивать файлы, финальный чек-лист). Нет субагентов, нет worktree, нет bundled-скриптов, нет MCP-инструментов. Поэтому он **переносится в Codex 1-в-1 без изменения поведения** — Codex читает те же инструкции и применяет их так же. Единственная дельта — **где скилл живёт и как триггерится** (плюс одна оговорка про сам артефакт `description`, см. ниже).

## Куда положить и как активировать

Codex автоматически читает `AGENTS.md` (корень проекта или `~/.codex/AGENTS.md`). Авто-триггера по `description` (как у Claude-скиллов) у Codex **нет** — поэтому решай одним из трёх способов:

1. **Правило в `AGENTS.md`** — самый надёжный путь. Готовый блок:

   ```md
   ## Написание скиллов
   Когда пользователь просит создать / написать / собрать новый скилл (или
   отредактировать существующий) — следуй инструкции skills/write-a-skill/SKILL.md:
   собери требования → набросай SKILL.md (+ REFERENCE/EXAMPLES/scripts по нужде)
   → прогони по review-чеклисту. Полный текст правил — там же.
   ```

2. **Кастомный промпт для слэш-вызова** — положи копию инструкции в `~/.codex/prompts/write-a-skill.md`, тогда вызов `/write-a-skill` в чате Codex подтянет её как промпт. (У Codex слэш-команды — это файлы в `~/.codex/prompts/`, а не авто-скиллы.)

3. **Вручную** — просто открой `SKILL.md` и попроси Codex действовать по нему.

Осмысленная фраза-триггер: «создай/напиши скилл». Под Codex её обрабатывает правило из `AGENTS.md` (вариант 1) либо слэш-промпт (вариант 2).

## Что меняется против Claude-версии

Только две вещи, и обе — про окружение, а не про сам процесс написания скилла:

1. **Активация.** В Claude скилл сам всплывает по `description`; в Codex — только через `AGENTS.md` / `~/.codex/prompts/` / руками (см. выше).

2. **Оговорка про поле `description` в создаваемом скилле.** Раздел «Description Requirements» в SKILL.md остаётся в силе **для целевых платформ, где авто-триггер есть** (Claude Code и совместимые). Но имей в виду: **в самом Codex это поле не запускает скилл автоматически** — там триггер обеспечивает запись в `AGENTS.md`. Поэтому, если новый скилл пишется в том числе под Codex, наряду с хорошим `description` сразу заготовь короткий блок-правило для `AGENTS.md` (как в этом файле выше). `description` от этого не теряет смысл — он по-прежнему обязателен и работает на Claude-стороне.

Всё остальное — структура папки, шаблон, правила «добавлять скрипты для детерминированных операций», «разбивать файл при >100 строк», ссылки на ресурсы на один уровень, review-чек-лист — применяется без изменений.

## Ресурсные файлы

Если создаваемый скилл выносит контент в `REFERENCE.md`, `EXAMPLES.md`, `*-FORMAT.md`, `scripts/`, `data/` — это работает в Codex так же: файлы лежат на диске, Codex читает их по ссылке из `SKILL.md`. Единственный нюанс — у Codex **нет авто-запуска bundled-скриптов**: если в создаваемый скилл кладётся утилита (`scripts/helper.js` и т.п.), Codex/пользователь вызывает её **явно** (прямой запуск или `codex exec "<команда>"`), а в `SKILL.md` стоит дать точную команду запуска. Guard для GPT: **нет нужного инструмента/команды — не выдумывай, остановись и сообщи пользователю.**
