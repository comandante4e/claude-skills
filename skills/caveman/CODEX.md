# Caveman — версия для Codex CLI (GPT под капотом)

Это Codex-адаптация скилла `caveman`; логика и процесс — те же, что в [SKILL.md](SKILL.md), здесь только дельта под Codex.

Скилл **чисто инструкционный**: один файл с правилами стиля ответа. Ни субагентов, ни инструментов, ни bundled-скриптов, ни ресурсных файлов он не использует. Поэтому **поведение переносится 1-в-1**: те же правила сжатия, та же persistence, те же исключения Auto-Clarity. Единственная дельта — **где живут правила и как они активируются** под Codex.

## Куда положить и как активировать

У Codex нет авто-триггера по `description` (как у Claude-скиллов). Вместо этого Codex автоматически читает `AGENTS.md`. Три рабочих варианта:

1. **Правило в `AGENTS.md`** (корень проекта или глобально `~/.codex/AGENTS.md`) — постоянно доступно, реагирует на кодовые фразы.
2. **Кастомный промпт `~/.codex/prompts/caveman.md`** — даёт слэш-вызов `/caveman` в сессии Codex. В файл кладёшь тело правил (содержимое SKILL.md без YAML-фронтматтера).
3. **Вручную** — вставить правила в первое сообщение сессии.

Минимальный блок для `AGENTS.md`:

```md
## Caveman mode (ультра-сжатые ответы)
Триггеры: «caveman mode», «talk like caveman», «use caveman», «меньше токенов»,
«be brief», `/caveman`. После триггера — отвечай терсно как умный пещерный человек:
выкидывай артикли (a/an/the), filler (just/really/basically), любезности
(sure/certainly/happy to), хеджирование. Фрагменты OK, короткие синонимы, сокращения
(DB/auth/config/fn). Причинность через стрелку (X -> Y). Технические термины, блоки кода
и тексты ошибок — БЕЗ изменений, точь-в-точь.
АКТИВНО каждый ответ, пока пользователь не скажет «stop caveman» / «normal mode».
Временно выходи из режима для: предупреждений о безопасности, подтверждений необратимых
действий, многошаговых последовательностей (порядок фрагментов критичен), просьбы уточнить.
Полная версия — skills/caveman/CODEX.md и skills/caveman/SKILL.md.
```

## Что меняется против Claude-версии

Поведенчески — **ничего**. Нет субагентов, worktree, MCP-инструментов и скриптов, которые требовали бы маппинга. Релевантны только две мелочи:

- **Слэш-вызов.** `/caveman` в Claude Code → кастомный промпт `~/.codex/prompts/caveman.md` под Codex (или просто кодовая фраза, обрабатываемая правилом из `AGENTS.md`).
- **Persistence через `AGENTS.md`.** Главная фишка скилла — режим держится «каждый ответ, пока не выключат». В Codex это особенно важно прописать явным правилом в `AGENTS.md`: правило в контексте каждого хода и удерживает режим, тогда как разовая вставка в одно сообщение может «утечь» по мере диалога. Если правила нет в `AGENTS.md`/промпте — Codex дрейфует обратно к обычному многословию.

Никаких гейтов, инструментов «нет команды — остановись» и параллелизма здесь нет — секции про worktree/субагентов неприменимы.
