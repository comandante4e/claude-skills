# Russian Titles — версия для Codex CLI (GPT под капотом)

Это Codex-адаптация скилла `russian-titles`; логика и процесс те же, что в [SKILL.md](SKILL.md) — все пользовательские заголовки на русском. Скилл **чисто инструкционный**: только правила оформления текста, без субагентов, скриптов и ресурсных файлов. Поведение переносится **1-в-1**. Единственные дельты — **где он живёт** под Codex и **на какие поля каких тулов** это правило ложится (у Codex другой набор инструментов).

## Куда положить и как активировать

Codex автоматически читает `AGENTS.md` (корень проекта или `~/.codex/AGENTS.md`). Авто-триггера по `description` (как у Claude-скиллов) у Codex **нет** — правило должно физически лежать в `AGENTS.md`, чтобы действовать.

Триггер-фразы у скилла нет — он «active at all times». Поэтому правильнее всего вставить его как **постоянное правило** в `~/.codex/AGENTS.md` (глобально на все проекты). Слэш-вызов `/russian-titles` через кастомный промпт в `~/.codex/prompts/russian-titles.md` тоже можно завести, но это разовое включение в сессии — для «всегда на русском» лучше глобальный `AGENTS.md`.

Готовый блок для `~/.codex/AGENTS.md`:

```md
## Заголовки — всегда на русском
Все пользовательские заголовки и подписи, которые я создаю, пишу по-русски:
сообщения git commit, заголовки/тело PR (`gh pr create --title/--body`),
заголовки H1/H2 в markdown-документах для пользователя, названия задач планировщика,
описания шагов в плане/TODO. Стиль: без эмодзи (если не попросили), 2–6 слов,
императив или существительное-фраза. Технические идентификаторы — имена файлов,
переменных, команд, флагов — остаются латиницей внутри русского заголовка.
```

## Что меняется против Claude-версии

Поведение — то же. Меняется только **карта тулов**: список из SKILL.md написан под инструменты Claude Code, у Codex их в таком виде нет. Маппинг полей, где правило по-прежнему применяется под Codex:

- **`git commit -m`, `gh pr create --title/--body`** — есть и в Codex, правило применяется без изменений (русский текст). Если коммитишь через `codex exec` — текст сообщения тоже на русском.
- **Запись markdown-документов для пользователя** (через правки файлов) — заголовки H1/H2 на русском. То же правило, что в SKILL.md.
- **`mcp__ccd_session__spawn_task` / `mcp__ccd_session__mark_chapter` / `TodoWrite`** — это инструменты Claude Code, **у Codex их нет**. Их прямые аналоги в Codex отсутствуют; ближайшее — внутренний план/TODO-список Codex и описания шагов: их формулировки тоже держи на русском.
- **`mcp__scheduled-tasks__create_scheduled_task`** — Claude-специфичный MCP. Если пользователь подключил планировщик как MCP-сервер в `~/.codex/config.toml` — название задачи на русском по тому же правилу; если такого тула нет — пункт неприменим.
- **`Bash` tool `description`** — у Codex отдельного поля `description` для команды нет (команды просто исполняются). Эквивалент — пояснение к команде, которое ты даёшь пользователю в чате: его веди по-русски.

То, что в SKILL.md помечено как «не контролирую» (имена скиллов/тулов, имена файлов в коде, авто-заголовок чата в UI), под Codex так же вне зоны влияния и латиница там не нарушает правило.

## Что переносится без изменений

- **Стиль** (без эмодзи, 2–6 слов, императив/существительное, англоязычные техтермины и идентификаторы латиницей внутри русского заголовка) — целиком из [SKILL.md](SKILL.md), раздел «Стиль».
- **Примеры** оформления (commit message, TODO-формулировки, заголовки документов) — применимы как есть; различаются только имена тулов-носителей.
- Отдельных ресурсных `.md`, каталогов `scripts/` или `data/` у скилла нет — переносить и запускать нечего.

## GPT-специфика

- Держи правило в `~/.codex/AGENTS.md`, а не полагайся на «он сам вспомнит» — без записи в `AGENTS.md` Codex правило не подхватит.
- Если поля/тула из списка SKILL.md в Codex нет — **не выдумывай аналог**, просто пропусти этот пункт; правило применяется только к реально существующим под Codex полям (commit, PR, markdown, план).
