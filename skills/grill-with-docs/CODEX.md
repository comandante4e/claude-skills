# grill-with-docs — версия для Codex CLI (GPT под капотом)

Это Codex-адаптация скилла `grill-with-docs`; логика и процесс те же, что в [SKILL.md](SKILL.md) — здесь только дельта под Codex.

Скилл **чисто инструкционный**: это протокол допроса (один вопрос за раз, вызов рекомендованного ответа, проверка терминов против глоссария, заострение размытых формулировок, сверка с кодом) плюс inline-правка документации (`CONTEXT.md`, ADR). Он не спавнит субагентов, не использует worktree, не тянет MCP-инструменты и **не несёт никаких скриптов** (в папке только `SKILL.md`, `CONTEXT-FORMAT.md`, `ADR-FORMAT.md`). Всё, что ему нужно от среды, — читать файлы (кодовую базу, существующие `CONTEXT.md` / `CONTEXT-MAP.md` / `docs/adr/`) и писать их. Поэтому **поведение переносится на Codex 1-в-1**; единственная реальная дельта — где скилл живёт и как триггерится.

## Куда положить и как активировать

Codex автоматически читает `AGENTS.md` (корень проекта или `~/.codex/AGENTS.md`). Авто-триггера по полю `description` (как у Claude-скиллов) у Codex нет — активацию нужно задать явно. Три способа:

1. **Правило в `AGENTS.md`** — Codex подхватит фразу-триггер в любой сессии этого проекта (или глобально через `~/.codex/AGENTS.md`).
2. **Кастомный промпт для слэш-вызова** — положи тело скилла в `~/.codex/prompts/grill-with-docs.md`, тогда сессию можно открывать командой `/grill-with-docs`.
3. **Вручную** — просто вставь содержимое `SKILL.md` в начало диалога с Codex.

Минимальный блок для `AGENTS.md` (осмысленная фраза вызова — «grill me against the docs» / «прожарь план против документации»):

```md
## grill-with-docs (допрос плана против доменной модели)
Когда пользователь говорит «прожарь план против документации» / «grill me with docs» —
веди безжалостный допрос по плану, по одному вопросу за раз, дожидаясь ответа на каждый;
к каждому вопросу давай свой рекомендованный ответ. Если на вопрос отвечает код —
вместо вопроса исследуй кодовую базу.
Сверяй термины с CONTEXT.md / CONTEXT-MAP.md (конфликт — называй сразу), заостряй
размытые термины до канонических, гоняй конкретные сценарии на граничные случаи,
сверяй заявления с кодом. По мере фиксации решений ОБНОВЛЯЙ документацию inline:
CONTEXT.md — по формату CONTEXT-FORMAT.md, ADR — по ADR-FORMAT.md (предлагай ADR
только если решение трудно откатить И неочевидно без контекста И это результат
реального трейд-оффа). Файлы создавай лениво — только когда есть что записать.
Полная версия — skills/grill-with-docs/CODEX.md и SKILL.md.
```

## Что меняется против Claude-версии

Поведенчески — **ничего**. Скилл не опирается на субагенты, Task, worktree, бандл-скрипты или специфичные Claude/MCP-инструменты, поэтому маппить нечего. Дельты только организационные:

- **Где живёт и как триггерится** — см. выше (`AGENTS.md` / `~/.codex/prompts/grill-with-docs.md` / ручная вставка), вместо авто-триггера по `description`.
- **Ресурсные файлы переносятся как есть.** [CONTEXT-FORMAT.md](CONTEXT-FORMAT.md) и [ADR-FORMAT.md](ADR-FORMAT.md) Codex читает с диска так же, как Claude. Держи их рядом со скиллом (или укажи путь в промпте), чтобы агент мог открыть формат в момент записи `CONTEXT.md` / ADR. Менять их под Codex не нужно.
- **Чтение и запись — напрямую.** Codex и так читает кодовую базу и правит файлы своими средствами; никакого авто-запуска вспомогательных скриптов тут нет и не требуется (скриптов в скилле нет). Inline-обновление `CONTEXT.md`/ADR по ходу допроса работает штатно.

## GPT-специфика (необязательно, но помогает)

- **Один вопрос за раз — держи жёстко.** GPT под нагрузкой склонен вываливать список вопросов сразу; правило «по одному, ждать ответ» сформулируй явно в промпте/`AGENTS.md`, иначе протокол ломается.
- **Не пиши ADR на автомате.** Гейт из трёх условий (трудно откатить · неочевидно без контекста · реальный трейд-офф) — соблюдай как чек-лист; не хватает хотя бы одного — ADR не предлагать.
- **Guard «не выдумывай».** Если `CONTEXT.md` / `CONTEXT-MAP.md` / `docs/adr/` нет — не сочиняй их содержимое из воздуха: создавай файл лениво, только когда в допросе реально закрепился первый термин/решение, и ровно по формату из ресурсных файлов.
