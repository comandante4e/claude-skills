# to-prd — версия для Codex CLI (GPT под капотом)

Это Codex-адаптация скилла `to-prd`; логика и процесс те же, что в [SKILL.md](SKILL.md) (синтез PRD из текущего контекста без интервью, шаблон PRD, публикация в трекер с меткой `ready-for-agent`). Здесь только дельта механики под Codex.

## Куда положить и как активировать

Codex автоматически читает `AGENTS.md` (корень проекта или `~/.codex/AGENTS.md`). Авто-триггера по `description` (как у Claude-скиллов) у Codex **нет** — нужно дать правило руками. Варианты:

- **Правило в `AGENTS.md`** — самый надёжный путь: Codex подхватит его в каждой сессии проекта.
- **Кастомный промпт `~/.codex/prompts/to-prd.md`** — даёт слэш-вызов `/to-prd` в интерактивной сессии (полный текст инструкции из SKILL.md внутри файла).
- **Вручную** — попросить Codex «собрать PRD из текущего контекста и опубликовать в трекер».

Минимальный блок для `AGENTS.md`:

```md
## to-prd (PRD из текущего контекста)
Когда пользователь просит собрать PRD из текущего контекста — НЕ интервьюируй,
синтезируй из того, что уже известно по диалогу и коду. Изучи репозиторий,
используй доменный глоссарий проекта, уважай ADR в затронутой области.
Набросай модули (ищи deep modules с простым тестируемым интерфейсом),
СВЕРЬ их и выбор покрытия тестами с пользователем. Затем напиши PRD по шаблону
из skills/to-prd/SKILL.md и опубликуй в трекер проекта с меткой `ready-for-agent`.
Словарь трекера и меток должен быть задан заранее (аналог /setup-matt-pocock-skills);
нет трекера/метки — остановись и спроси, не выдумывай.
Полная версия — skills/to-prd/CODEX.md.
```

## Что меняется против Claude-версии

Сам по себе скилл почти полностью **инструкционный** (правила синтеза + шаблон PRD) — текст переносится 1-в-1, поведение не меняется. Дельты Codex касаются только двух внешних зацепок: предустановки контекста и публикации в трекер.

### 1. Слэш-вызов и предусловие `/setup-matt-pocock-skills`
- `/to-prd` в Claude Code → в Codex это промпт-файл `~/.codex/prompts/to-prd.md` или правило в `AGENTS.md`.
- SKILL.md ссылается на `/setup-matt-pocock-skills`, который заранее кладёт в контекст **словарь трекера и набор triage-меток**. У Codex этой слэш-команды нет — словарь трекера/меток нужно занести в `AGENTS.md` проекта заранее (или передать в запросе). Если он не задан — **остановись и сообщи**, не угадывай ни тип трекера, ни вокабуляр меток.

### 2. Публикация в issue tracker → MCP / CLI, который пользователь подключает сам
Финальный шаг — создать issue в трекере и навесить метку `ready-for-agent`. У Codex нет встроенного инструмента трекера; используется то, что настроено в `~/.codex/config.toml`:
- **GitHub Issues** — `gh issue create --label ready-for-agent ...` через Bash (если `gh` установлен и авторизован), либо GitHub MCP.
- **Linear / Jira / Notion и т.п.** — соответствующий MCP-сервер из `~/.codex/config.toml`.
- Метка `ready-for-agent` должна существовать в трекере заранее. Нет настроенного трекера или метки — **не имитируй публикацию**: сохрани PRD локально (например, Markdown-файл) и сообщи пользователю, что нужно подключить трекер.

### 3. Сверка с пользователем — без субагентов
Скилл просит свериться по модулям и по выбору покрытия тестами. Это обычный диалог в той же сессии Codex; **спавна субагентов / свежего контекста на шаг тут нет и не требуется** (в отличие от harness-скиллов). Никакого worktree, параллели или bundled-скриптов в этом скилле нет.

### 4. GPT-специфика
- Иди по шагам SKILL.md явно: (1) изучить репозиторий и глоссарий/ADR → (2) набросать и **сверить** модули + покрытие тестами → (3) написать PRD по шаблону и опубликовать. Не перепрыгивай шаг сверки модулей.
- Не вставляй в PRD пути к файлам и код (как и требует SKILL.md) — исключение только для snippet'а, кодирующего решение (state machine / reducer / schema / type shape).
- Для воспроизводимой публикации из скрипта/CI можно использовать `codex exec "<задача>"`, но интерактивная сверка модулей с пользователем удобнее в обычной сессии.

## Ресурсные файлы

Дополнительных ресурсов (REFERENCE.md, *-FORMAT.md, `scripts/`, `data/`) у скилла нет — весь PRD-шаблон встроен в [SKILL.md](SKILL.md) и читается Codex'ом с диска так же, как Claude.
