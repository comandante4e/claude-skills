# Excel: формулы для производных значений

## Главное правило

**Любая ячейка, чьё значение зависит от других ячеек, записывается формулой, а не вычисленным числом.**

```
❌ Плохо: ячейка C1 = 150  (где 150 = 100 + 50, посчитано в коде)
✅ Хорошо: ячейка C1 = "=A1+B1"
```

Причина: пользователь должен иметь возможность поменять входное значение в Excel — и сразу увидеть обновлённые производные. Если я вкатал результат числом, он мёртвый, и таблица превращается в одноразовый артефакт.

## Декомпозиция перед записью

Прежде чем писать любую ячейку с числом, спроси себя:

1. **Это вход (variable)?** — пользователь будет это менять, остальное от него зависит. Пиши литералом.
2. **Это выход (derived)?** — это результат вычисления над другими ячейками. Пиши формулой со ссылками.
3. **Это константа (константа)?** — типа `НДС = 20%`, `курс = 95`. Лучше вынести в отдельную ячейку с именем (named range) или в шапку, и ссылаться формулой.

**Если не уверен — спроси пользователя.** Не угадывай. Используй `AskUserQuestion` с конкретным списком ячеек: какие из них вход, какие выход, какие константы.

## Когда спрашивать (триггеры для AskUserQuestion)

- Пользователь дал готовый набор цифр без пометок — неясно, что из этого «исходник», а что результат.
- В таблице есть колонки типа «Итого», «Сумма», «%», «Маржа», «Разница», «Среднее» — почти наверняка derived, но подтверди.
- Есть подозрение на промежуточные вычисления, которые лучше вынести в отдельные ячейки (DRY).
- Пользователь скидывает CSV с готовыми числами и просит «сделай красиво в xlsx» — спроси, какие колонки рассчитываемые.

Формат вопроса: дай конкретный список ячеек/колонок с пометками `[вход/выход/константа?]`, пусть пользователь подтвердит или поправит.

## Технические нюансы (openpyxl и аналоги)

### openpyxl

```python
ws["C1"] = "=A1+B1"          # формула
ws["A1"] = 100                # литерал (вход)
ws["B1"] = 50                 # литерал (вход)
```

**Важно:** openpyxl записывает формулу как строку, но Excel её исполнит при открытии. Если нужно показать пользователю результат сразу (например, читая обратно через openpyxl), грузить файл с `data_only=True` — но это покажет последнее сохранённое значение Excel, а не пересчитанное.

### Локализация формул

- В русской версии Excel функции переводятся: `SUM` → `СУММ`, `IF` → `ЕСЛИ`, `AVERAGE` → `СРЗНАЧ`, и т.д.
- **При записи через openpyxl всегда пиши английскими именами** (`=SUM(A1:A10)`). Excel сам переведёт их в UI в зависимости от языка установки.
- Разделитель аргументов: в файле, который пишет openpyxl, — запятая `,` (US-стандарт). Excel при открытии покажет точку с запятой `;`, если локаль такая, но это косметика.
- Десятичный разделитель внутри формулы — всегда точка `.` (`=A1*0.2`), даже в русской локали.

### Ссылки

- Относительные: `A1` — при копировании формулы вниз/вбок ссылка смещается.
- Абсолютные: `$A$1` — не смещается. Использовать для констант (курс, ставка налога).
- Смешанные: `$A1` / `A$1` — фиксируется только колонка или только строка.

**Правило:** ссылки на ячейки-константы (налог, курс, коэффициенты) пиши абсолютными — `$B$1` — чтобы при автозаполнении вниз не съезжали.

### Именованные диапазоны

Для частоиспользуемых констант лучше создать named range:

```python
from openpyxl.workbook.defined_name import DefinedName
wb.defined_names["НДС"] = DefinedName("НДС", attr_text="Параметры!$B$2")
ws["D5"] = "=C5*(1+НДС)"
```

Формулы становятся читаемыми, и пользователю проще найти, где менять.

### Промежуточные вычисления

Не зашивай длинные цепочки в одну формулу — выноси промежуточные шаги в отдельные ячейки:

```
❌ Одна ячейка: =((A1*B1)-(A2*B2))/((A1*B1)+(A2*B2))*100
✅ Разнести:
   D1 = =A1*B1       ; «Выручка_2024»
   D2 = =A2*B2       ; «Выручка_2023»
   D3 = =D1-D2       ; «Дельта»
   D4 = =D1+D2       ; «Сумма»
   D5 = =D3/D4*100   ; «Рост, %»
```

Так пользователь видит логику и может проверить каждый шаг.

## Типичные паттерны

### Колонка «Итого» сбоку

```python
for row in range(2, 12):
    ws[f"D{row}"] = f"=SUM(A{row}:C{row})"
```

Не считай суммы в Python и не вписывай готовое число — это убивает live-обновление.

### Строка «Итого» снизу

```python
ws["A12"] = "Итого"
ws["B12"] = "=SUM(B2:B11)"
ws["C12"] = "=SUM(C2:C11)"
```

### Процент от итога

```python
ws["E2"] = "=B2/$B$12"   # B12 — абсолютная ссылка на итог
# и формат ячейки — процентный
ws["E2"].number_format = "0.00%"
```

### VLOOKUP / справочник

Если есть отдельный лист со справочником (курсы, тарифы) — ссылайся через VLOOKUP/XLOOKUP, не зашивай значения:

```python
ws["C2"] = "=VLOOKUP(B2, Справочник!$A$2:$B$100, 2, FALSE)"
```

## Чеклист перед коммитом xlsx

- [ ] Все «Итого», «Сумма», «Средн.», «%», «Разница» — формулами
- [ ] Константы вынесены в отдельные ячейки (или named range), формулы ссылаются на них
- [ ] Абсолютные ссылки `$...$` там, где это нужно при автозаполнении
- [ ] Функции на английском (`SUM`, не `СУММ`)
- [ ] Десятичный разделитель — точка
- [ ] Длинные формулы разбиты на промежуточные шаги
- [ ] Если был сомнительный момент — задал вопрос пользователю до записи

## Когда литерал допустим (исключения)

- **Снапшот / отчёт «как есть на дату X».** Если пользователь явно сказал «зафиксируй цифры, не пересчитывать» — пиши литералами и в шапке листа отметь дату фиксации.
- **Импортированные сырые данные.** Если копируешь CSV → xlsx без вычислений — литералы норм.
- **Текстовые метки, ID, даты-факты** — это не «производные», это сами по себе данные.

Во всех остальных случаях — формула.
