# Excel: формулы для производных значений — версия для Codex CLI (GPT под капотом)

Это Codex-адаптация скилла `excel-formulas`; логика и правила — **те же, что в [SKILL.md](SKILL.md)** (главное правило «производная ячейка = формула, а не число», декомпозиция вход/выход/константа, локализация формул, чеклист, исключения). Скилл **чисто инструкционный** — он не спавнит субагентов, не использует worktree, MCP и bundled-скрипты. Поэтому переносится **1-в-1 без изменения поведения**; дельта под Codex — только в том, **где он живёт**, **как триггерится** и чем заменить один Claude-инструмент.

## Куда положить и как активировать

У Codex **нет авто-триггера по `description`** (как у Claude-скиллов). Codex автоматически читает `AGENTS.md`. Варианты установки:

- **Правило в `AGENTS.md`** (корень проекта или `~/.codex/AGENTS.md` глобально) — основной способ: правило будет действовать постоянно, без явного вызова.
- **Кастомный промпт в `~/.codex/prompts/excel-formulas.md`** — для слэш-вызова `/excel-formulas` в сессии Codex, когда хочешь подгрузить правила точечно.
- **Вручную** — вставить текст SKILL.md в начало диалога / в ChatGPT перед задачей с таблицей.

Поскольку у скилла нет кодовой фразы, а триггер — «работаю с .xlsx/.csv и считаю производные значения», лучший вариант для Codex — короткое **постоянное** правило в `AGENTS.md`:

```md
## Excel: формулы, а не числа
При создании/правке .xlsx/.xlsm/.xls/.csv любое значение, зависящее от других
ячеек (Итого, Сумма, %, Маржа, Разница, Среднее и т.п.), пиши ЖИВОЙ ФОРМУЛОЙ
(=A1*B1, =SUM(...), =VLOOKUP(...)), а не вычисленным числом. Входы — литералами,
константы (НДС, курс) — отдельной ячейкой/named range с абсолютной ссылкой.
Функции пиши по-английски (SUM, не СУММ), десятичный разделитель — точка.
Если неясно, какие ячейки вход, какие выход, какие константы — СПРОСИ до записи.
Полные правила и чеклист — skills/excel-formulas/SKILL.md.
```

## Что меняется против Claude-версии

### 1. `AskUserQuestion` → обычный вопрос текстом
SKILL.md в момент сомнения предлагает звать инструмент `AskUserQuestion` (структурированный опрос Claude Code). У Codex такого инструмента **нет**. Маппинг: задай тот же вопрос **обычным текстом в чате** и дождись ответа до записи файла. Содержание не меняется — дай конкретный список ячеек/колонок с пометками `[вход / выход / константа?]`, пусть пользователь подтвердит или поправит. Guard прежний: **не уверен — не угадывай, остановись и спроси.**

### 2. Python/openpyxl-примеры — не bundled-скрипты, запуск явный
Блоки `ws["C1"] = "=A1+B1"`, named range, циклы по строкам — это **иллюстрации приёма**, а не скрипты в `scripts/`, которые харнесс запускает сам (у скилла нет каталогов `scripts/`/`data/`, а у Codex и нет авто-запуска вспомогательных скриптов). Если по ходу задачи реально генеришь .xlsx кодом — Codex пишет и **явно запускает** Python сам (`python script.py`) или через `codex exec "..."` для воспроизводимого прогона; пользователь подтверждает запуск/устанавливает `openpyxl` сам.

### 3. Ресурсных .md у скилла нет
Скилл состоит из одного `SKILL.md` — отдельных REFERENCE.md/*-FORMAT.md нет. Если бы были, Codex читал бы их так же, с диска. Все правила локализации, чеклист и исключения остаются внутри SKILL.md и применяются дословно.

### 4. GPT-специфика
- Перед записью каждой числовой ячейки прогоняй явный чек: **вход / выход / константа** (шаг из SKILL.md) — не «на глаз».
- Перед сохранением .xlsx пройди **чеклист из SKILL.md** по пунктам, не пропуская: формулы для всех итогов/%, абсолютные `$...$` где нужно автозаполнение, английские имена функций, точка как десятичный разделитель, длинные формулы разбиты.
- `codex exec` — для воспроизводимой генерации/перегенерации таблицы одним прогоном.
