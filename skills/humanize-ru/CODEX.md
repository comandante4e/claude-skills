# Очеловечивание текста (RU) — версия для Codex CLI (GPT под капотом)

Это Codex-адаптация скилла `humanize-ru`. Логика и процесс — **те же, что в [SKILL.md](SKILL.md)**: шесть техник, словарь русских ИИ-маркеров, чек-лист самопроверки и принцип «не выдумывай». Здесь только дельта под Codex.

Скилл **чисто инструкционный**: это набор правил и текстовых трансформаций. Нет субагентов, нет вызова инструментов, нет MCP, нет bundled-скриптов (`scripts/`, `data/` отсутствуют). «Шаблоны промптов» в [TECHNIQUES.md](TECHNIQUES.md) — это текст, который модель применяет к черновику сама, а не вызовы внешних команд. Поэтому **поведение переносится на Codex 1-в-1**. Единственная дельта — **где скилл живёт и как триггерится**, потому что авто-триггера по `description` (как у Claude-скиллов) у Codex нет.

## Куда положить и как активировать

Codex автоматически читает `AGENTS.md`. Подключить скилл можно тремя способами:

1. **Правило в `AGENTS.md`** (в корне проекта или глобально `~/.codex/AGENTS.md`) — Codex увидит его в каждой сессии и будет очеловечивать текст по запросу.
2. **Кастомный промпт для слэш-вызова** — положи файл в `~/.codex/prompts/humanize-ru.md` (содержимое — текст из SKILL.md, либо короткая отсылка к нему + правила). Тогда скилл вызывается в чате как `/humanize-ru`.
3. **Вручную** — просто скопируй нужную технику/чек-лист в чат или вставь в ChatGPT.

Сами файлы `SKILL.md`, `AI-MARKERS.md`, `TECHNIQUES.md` переносятся **как есть** и читаются Codex'ом с диска так же, как Claude'ом (`@AI-MARKERS.md`, `@TECHNIQUES.md` или просто чтение по пути). Ссылки между ними работают одинаково.

Минимальный блок для `AGENTS.md`:

```md
## Очеловечивание русского текста (humanize-ru)
Когда пользователь пишет/правит русский текст (пост, статья, письмо, рассылка,
лендинг, заметка, сценарий) и просит «убрать запах ИИ / нейросетевой стиль»,
«очеловечить», «переписать под мой стиль», «убрать клише и воду», «настроить ритм»,
humanize / de-AI — работай по skills/humanize-ru/SKILL.md:
выбери технику(и) из шести, правь форму а не факты, прогони чек-лист из AI-MARKERS.md.
Главное правило: НИЧЕГО НЕ ВЫДУМЫВАЙ — нет факта/кейса/цифры, спроси пользователя
или оставь [плейсхолдер]. Только для русского; для английского — отдельный скилл.
```

## Что меняется против Claude-версии

Ничего по сути. Скилл не использует субагентов, `git worktree`, MCP-инструменты или вспомогательные скрипты — переносить или маппить нечего. Применима только GPT-специфика по дисциплине:

- **Явно держи чек-лист.** После правки прогони список из [AI-MARKERS.md](AI-MARKERS.md) пунктом за пунктом; GPT охотно «причёсывает» текст и сам же возвращает ИИ-маркеры — не пропускай самопроверку.
- **Guard «не выдумывай» — критичнее.** Добавляя личный опыт, цифры, цитаты или детали, которых нет в исходнике, не сочиняй: спроси пользователя или оставь явный `[плейсхолдер]` (техники 4 и 5). Живой стиль ≠ выдуманные факты.
- **Воспроизводимый прогон.** Для пакетной чистки нескольких файлов одним правилом удобен `codex exec "очеловечь по skills/humanize-ru/SKILL.md: <файл>"` — неинтерактивно, без накопления истории между файлами.
